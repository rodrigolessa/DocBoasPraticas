Conhecimentos em Design Patterns (padrões de projeto)
1)	O que é um padrão?
R: Um padrão é uma solução reutilizável que pode ser comumente aplicada em problemas que ocorrem no projeto de software.
 
2)	Cite alguns padrões que você conhece.
R: Criacionais (Constructor, Factory, Abstract, Prototype, Singleton, Builder, Factory, Unit of Work)
  Estruturais (Decorator, Facade, Flyweight, Adapter, Proxy)
  Comportamentais (Iterator, Mediator, Observer, Visitor)

3)	Para que serve o padrão singleton?
R: Para criar uma classe não-estática que permite que apenas uma instância de si mesma seja criada.

4)	Considerando o modelo Model-View-Controller, descreva o Controller:
R: A camada é responsável por fazer o controle de fluxo da aplicação, as requisições são feitas nas ActionsResulta que são implementadas no Controller, que é responsável por conhecer as demais camadas, em uma arquitetura baseada em DDD; podendo apenas ele acessar e repassar as informações que a View necessita.
R: Envia comando para o modelo para atualização de estados e também envia comandos para a view para atualizar a apresentação em relação a mudança no modelo.
